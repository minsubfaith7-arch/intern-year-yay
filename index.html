Set up your schedule

This page renders your live QGenda schedule. Two ways to get it in:

RecommendedLive sync via GitHub

  1. In QGenda, copy your personal calendar-sync (iCal) link.
  2. On GitHub, use this repo as a template (or fork it).
  3. Add the link as an Actions secret named QGENDA_ICS_URL.
  4. Run the Sync QGenda feed workflow once, then enable GitHub Pages.

Full walkthrough in the README. Your iCal link (which contains a private key) stays in the secret — it is never published.

InstantImport a .ics file

Export/download your QGenda calendar as an .ics file and drop it here. It stays in this browser only (localStorage) — nothing is uploaded.

Imports are a snapshot — they don't update when QGenda changes. Or preview with demo data ↗

    Block by block

    Each card shows a condensed rotation guide plus the interns and residents scheduled on the same rotation code around your dates. A small date range on a name means partial overlap — residents flip on an offset cycle, so some overlap only a few days at the edges.

    Fine print

    • Your schedule is live; everyone else's isn't. Your blocks, dates and the day-level grid come straight from your QGenda feed. The "on with you / on before you" rosters come from a static, program-wide QGenda export printed 7/8/2026 — swaps and reassignments made after that date won't show here.
    • Rosters match on the exact rotation code. Night coverage usually runs under separate codes (e.g. "SFW Night Intern", "SF ICU Night"), so night-float trainees aren't included in ward-block rosters, and vice versa.
    • The Rotation guide dropdowns condense the site "Ultimate Guides" (Parnassus 6/2026, SFGH, VA) and Max's block notes — they describe last year's structures, so times, caps and rooms may shift. Door codes, shared logins and personal cell numbers were deliberately left off this public page (they're in the source guides on Box).
    • If this page is public (GitHub Pages), your day-level schedule is public too. Your QGenda iCal URL itself stays in a repo secret and is never exposed.
    • You're viewing a manually imported .ics snapshot — it won't refresh on its own. Re-import to update, or set up live sync. Clear imported data.
